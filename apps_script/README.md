# Apps Script 外部設計

このディレクトリの `main.js` は、Schronu の出力を貼り付けて使う Google Spreadsheet 向けの補助スクリプトです。

## 対象シート

対象シートは以下の2つです。

- `実ログ`
- `優先度低い順`

両シートは、同じタスク集合を異なる順序で表示する用途を想定しています。行番号ではなく、B列の `task_id` をキーにして同一タスクを対応付けます。

## 列仕様

| 列 | 用途 |
| --- | --- |
| B | `task_id`。両シート間の同期キー |
| K | 同期対象 |
| M | 同期対象 |
| O | 同期対象 |
| P | 同期対象 |

データ行は3行目から始まる前提です。

## 起動時の動作

Google Sheets を開くと、メニューに `ユーザー関数` が追加されます。

メニュー項目は以下です。

- `時刻形式を再適用`

また、起動時にも時刻形式の再適用を実行します。

## 時刻形式の再適用

`実ログ` と `優先度低い順` の以下の範囲に、時刻表示形式 `hh:mm` を設定します。

- `K3:L500`
- `N3:O500`

対象シートが存在しない場合は、存在しないシート名をアラートで表示します。

## 編集時の同期

`実ログ` または `優先度低い順` の3行目以降で、K/M/O/P列を編集した場合、もう一方のシートから同じ `task_id` の行を探し、同じ列へ値を反映します。

例:

- `実ログ` のB列が `abc` の行でK列を編集する
- `優先度低い順` のB列が `abc` の行を探す
- 見つかった行のK列へ同じ値を設定する

相手シートに同じ `task_id` が存在しない場合は何もしません。B列が空の行も対象外です。

## 貼り付け時の扱い

B列を含む複数行貼り付けは、`shell/copy_for_spreadsheet.sh` の出力を貼り直した操作とみなします。

この場合は同期も復元も行いません。丸ごと貼り直す操作によって、もう一方のシートの値が意図せず変更されることを避けます。

## 同時編集対策

編集時の同期処理では `LockService.getDocumentLock()` を使用します。ロックを取得できない場合、その編集イベントでは同期処理を行いません。

## 注意事項

- 同期は行番号ではなくB列の `task_id` で行います。
- 対象シート名、同期対象列、データ開始行は `main.js` の `SCHRONU_CONFIG` で管理します。
- Google Apps Script の `onEdit` イベントとして動作するため、最終的な動作確認は Google Sheets 上で行ってください。
